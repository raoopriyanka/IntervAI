🧩 2. “How do you debug backend errors?”
✅ Answer:

“I start by checking server logs to identify the exact error. Then I isolate whether the issue is in request data, business logic, or external dependencies. In my project, I used FastAPI logs and Swagger UI to test endpoints and narrow down the issue.”

🧠 3. “What is a 500 Internal Server Error?”
✅ Answer:

“A 500 error indicates a server-side failure where the request reaches the backend, but something goes wrong during processing, such as a code exception or dependency issue.”

🔐 4. “What issue did you face with passlib?”
✅ Answer:

“passlib was internally using bcrypt, but due to version incompatibility, it caused an AttributeError. Instead of forcing versions, I simplified the implementation by directly using bcrypt, which made the system more stable.”

🧱 5. “What did you learn from that bug?”
✅ Answer:

“I learned to rely on logs instead of guessing, and also understood how dependency mismatches can break applications. It improved my debugging approach significantly.”
